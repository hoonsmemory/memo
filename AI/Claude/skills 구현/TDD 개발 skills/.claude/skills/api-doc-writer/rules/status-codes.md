# rules/status-codes.md

## 기본 상태코드 컨벤션
- 200 OK: 조회 성공 + 본문 포함
- 201 Created: 생성 성공 + Location 헤더 포함(리소스 조회 경로)
- 204 No Content: 성공했으나 본문 없음(가입/변경 등)
- 400 Bad Request: 요청 검증 실패, 비즈니스 규칙 위반(유일성 포함), 잘못된 자격 증명(email/password 불일치 포함)
- 401 Unauthorized: 인증 토큰 없음/유효하지 않음
- 403 Forbidden: 인증은 됐으나 역할/권한 불일치
- 404 Not Found: 리소스 없음, 또는 소유권 없는 리소스 은닉(정책으로 선택)

## 참고 규칙
- "존재하지 않는 이메일" + "비밀번호 불일치"를 400으로 둘지(예시처럼), 401로 둘지 프로젝트 컨벤션이 갈린다.
  - 기본값: 400 (예시 준수)
  - 사용자 지정이 있으면 그에 맞춘다.
